# Тренировка работы с Apache Kafka

**docker-compose.yml**:<br>
поднимает docker контейнер с kafka и связанный с ним контейнер с kafka UI.

**main.py**:<br>

1. Создает в kafka топик `TOPIC_NAME`, в ней `NUM_CONSUMERS` партиций
2. Запускает продюсера, который с периодом `MSG_PAUSE` секунд отправляет сообщение в топик `TOPIC_NAME`.<br>
В сообщении есть ключ `key` вида `"hash-{number}"`, где<br>
`number = user_number * 2`, а `user_number` - случайное число в диапазоне `range(100)`.
3. Запускает `NUM_CONSUMERS` консюмеров в группе `GROUP_ID` и подписывает их на топик `TOPIC_NAME`. Kafka вешает каждого консюмера на одну партицию.

Константы, определяющие поведение:<br>
```python
TOPIC_NAME = 'my-test-topic'    # имя топика
GROUP_ID = 'my-test-group'      # имя группы
NUM_CONSUMERS = 3               # количество консюмеров и партиций
MSG_PAUSE = 3                   # Пауза между сообщениями в продюсере
```

## Подготовка:
Я использовал python 3.12.5. Должно работать в условном python 3.9, но ничего не могу гарантировать. Инструкция для Windows 10, линуксоиды, я думаю, и сами переведут их на `bash`. Для неопытных в linux - пользуйтесь инструкциями для `git bash`


Нужно создать и активировать виртуальное окружение python, а также установить нужные модули. Это можно сделать в любимом IDE или в терминале. Терминалом может выступать `git bash` (мое предпочтение), `cmd`, `PowerShell` или любой другой терминал, в котором вы привыкли работать. Далее идут объяснения для терминала. Возможно в вашем `cmd` надо поменять слеш `/` на обратный слеш `\`, но у меня и так всё работает.
### В папке проекта создайте виртуальное окружение python:
```cmd
path/to/python -m venv .venv
```
где:<br>
`path/to/python` - путь до exe-файла python нужной версии.<br>
`venv` - имя модуля виртуального окружения<br>
`.venv` - папка виртуального окружения
### Запустите виртуальное окружение
для `git bash`<br>
```bash
source .venv/Scripts/activate<br>
```
для `cmd`<br>
```cmd
.venv/Scripts/activate.bat<br>
```
для PS<br>
```PowerShell
.venv/Scripts/activate.ps1<br>
```
### Установка библиотек
```bash
pip install confluent_kafka
```

## Запуск:
1. В папке проекта запустить kafka в docker. Для этого требуется запущенный Docker Desktop:
```bash
docker-compose up -d
```
Он запустит два контейнера - kafka и KafkaUI

2. Запустить сервисы продюсеров и консюмеров:
```bash
python main.py
```


Рекомендую kafka запустить в одном терминале, а сервисы в другом. Это удобно, если вам нужно следить за состоянием запущенных контейнеров и одновременно просматривать в терминале логи сервисов.
Каждый продюсер/консюмер выводит в терминал лог о том, какое сообщение он в данный момент отправил/принял. Если сервисы запущены через `main.py`, все логи выводятся в один терминал вперемешку. Если вам хочется 
следить за логами каждого сервиса отдельно, запустите каждый сервис в отдельном терминале. В этом случае, естественно, main.py запускать не надо. Правда придется вручную создавать партиции.

## Остановка
1. Для остановки сервисов нажмите `Ctrl-C` в терминале, где запущены сервисы
2. Для остановки kafka введите `docker-compose down` в терминале

## Offset Explorer
Дополнительно можно запустить Offset Explorer и просматривать сообщения через него. Но если честно, мне гораздо больше понравилось работать с Kafka UI. Он сообщения читает в реальном времени, а Offset Explorer, только по нажатию кнопки.


### Настройка соединения Offset Explorer 3.0.3:
1. Меню File -> Add new connection или нажмите кнопку "Add cluster". 
2. Откроется окно настройки нового подключения.<br>
**Вкладка Properties**:<br>
*Блок General*<br>
&nbsp;&nbsp;`Cluster name` - любое имя, например `Local-Kafka`<br>
&nbsp;&nbsp;`Bootstrap servers` - `host.docker.internal:9092`, здесь после ввода можно сразу нажать кнопку `Ping` для проверки соединения.<br>
&nbsp;&nbsp;`Kafka Cluster Version` - 3.0 или выше, т.к. используется последний образ kafka. Я выбрал максимальный, 3.7<br>
*Блок Zookeeper*<br>
&nbsp;&nbsp;`Host` и `Port` оставить пустым, здесь используется KRaft и Zookeeper нам не нужен<br>
**Вкладка Security**:<br>
&nbsp;&nbsp;`Broker Security` -> `Type` - `Plaintext`<br><br>
3. Внизу окна есть кнопка `Test`, нажмите ее, чтобы протестировать соединение. Если всё окей, то вам предложат добавить это соединение, нажмите `Yes`.<br>
Слева есть дерево кластеров, в нём теперь появится ваш кластер `Local-Kafka`.

# qa-trainer-kafka
qa-trainer-kafka

Подготовка:
Я использовал python 3.12.5. Я не проверял работу на более ранних версиях, 
поэтому ничего не могу гарантировать.
Нужно создать и активировать виртуальное окружение python, а также установить 
нужные модули. Это можно сделать в любимом IDE или в терминале. Терминалом 
может выступать git bash (мое предпочтение), cmd, PowerShell или любой 
другой терминал в котором вы привыкли работать. Дальше идут объяснения для 
терминала.
1. В папке проекта создайте виртуальное окружение python:
path\to\python -m venv .venv
path\to\python - путь до exe-файла python нужной версии.
venv - имя модуля виртуального окружения
.venv - папка виртуального окружения
2. Запустите виртуальное окружение
git bash
source .venv/Scripts/activate
cmd
.venv/Scripts/activate.bat
PS
.venv/Scripts/activate.ps1
3. Установка библиотек
pip install confluent_kafka

Запуск:
1. В папке проекта запустить kafka в docker. Для этого требуется запущенный 
Docker Desktop:
docker-compose up -d
Он запустит два контейнера - kafka и KafkaUI

2. Запустить сервисы продюсеров и консюмеров:
python main.py

Рекомендую kafka запустить в одном терминале, а сервисы в другом. Это 
удобно, если вам нужно следить за состоянием запущенных контейнеров и 
одновременно просматривать в терминале логи сервисов. 
Каждый продюсер/консюмер выводит в терминал лог о том, какое 
сообщение он в данный момент отправил/принял. Если сервисы запущены через 
main.py, все логи выводятся в один терминал вперемешку. Если вам хочется 
следить за логами каждого сервиса отдельно, запустите каждый сервис в 
отдельном терминале, например
python producer.py
python consumer.py
В этом случае, естественно, main.py запускать не надо.

Дополнительно можно запустить Offset Explorer и просматривать сообщения 
через него. Настройка соединения Offset Explorer 3.0.3:
1. Меню File -> Add new connection или нажмите кнопку "Add cluster". 
2. Откроется окно настройки нового подключения. 
На вкладке Properties:
Блок General
Cluster name - любое имя, например Local-Kafka
Bootstrap servers - host.docker.internal:9092, здесь после ввода можно 
   сразу нажать кнопку Ping для проверки соединения.
General -> Kafka Cluster Version - 3.0 или выше, т.к. используется 
   последний образ kafka. Я выбрал максимальный, 3.7

Блок Zookeeper
Host и Port оставить пустым, здесь используется KRaft и Zookeeper нам не нужен

На вкладке Security:
Broker Security -> Type - Plaintext

Внизу окна есть кнопка Test, нажмите ее, чтобы протестировать соединение. 
Если всё окей, то вам предложат добавить это соединение, нажмите Yes.
3. Слева есть дерево кластеров, в нём теперь появится ваш кластер Local-Kafka


Остановка:
1. Для остановки сервисов нажмите Ctrl-C
2. Для остановки kafka:
docker-compose down
